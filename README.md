# P3-2-prompt-injection-and-ai-abuse-cases

> **Current Status:** Planning stage. This repository is intended for a future AI security project and is not yet a completed implementation.

## Overview
This repository is planned as the second project in Portfolio 3 (P3), focused on prompt injection, AI-specific abuse cases, and security issues that can arise in AI-enabled applications.

The purpose of this planned project is to study and organize examples of how AI systems may be misused, manipulated, or exposed to unsafe behavior through:
- prompt injection
- indirect instruction manipulation
- unsafe tool access
- data exposure attempts
- system prompt leakage
- insecure handling of model output

## Purpose
This repository is intended to serve as a structured space for a future AI security project focused on documenting and organizing AI abuse cases that are relevant to modern AI-enabled applications.

The long-term goal is to use this repository to better understand how attackers, malicious inputs, unsafe integrations, or weak controls could influence the behavior of AI systems such as:
- internal AI assistants
- document-based chat systems
- agentic AI tools
- AI help desk workflows
- AI-powered enterprise assistants
- retrieval-augmented generation (RAG) systems

## Planned Scope
This project is expected to focus on the following areas:

- prompt injection examples
- indirect prompt injection concepts
- AI abuse case documentation
- unsafe tool or action scenarios
- data leakage attempts
- insecure output handling
- defensive considerations and mitigations
- safe testing notes
